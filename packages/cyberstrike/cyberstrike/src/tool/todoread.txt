Read the current pentest operation plan for this session. This returns your active attack roadmap — all pending tests, in-progress exploits, completed findings, and cancelled items.

Use this tool PROACTIVELY and FREQUENTLY. Your todo list is your operational memory. Without it, you lose track of discovered attack surface, confirmed vulnerabilities awaiting escalation, and planned test cases.

## When to Use

- **Every conversation start**: Read todos before doing anything else. Resume where you left off.
- **After reconnaissance**: Check if new findings overlap with or invalidate existing todos.
- **After completing a test**: Read to pick the next highest-priority item.
- **After web context updates**: Cross-reference new proxy traffic (endpoints, credentials, roles) with existing todos.
- **After vulnerability confirmation**: Read to plan escalation — confirmed vulns should trigger new high-priority todos via todowrite.
- **When deciding what to do next**: The todo list drives your prioritization. High-priority pending items first.
- **Every 3-5 tool calls**: Stay synchronized. The todo list may have been updated by parallel operations.

## What You Get Back

A list of todo items, each with:
- **content**: The specific test or action to perform
- **status**: pending (queued), in_progress (active), completed (done), cancelled (dropped)
- **priority**: high (exploit chains, confirmed vulns, auth bypass), medium (potential vulns, stack-specific tests), low (informational, hardening)

## Usage

This tool takes NO parameters. Leave the input blank or empty. DO NOT include a dummy object, placeholder string, or a key like "input". LEAVE IT BLANK.
